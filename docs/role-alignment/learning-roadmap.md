# Learning Roadmap

## Overview

This roadmap outlines a phased approach to developing the technical skills required for an Amazon IT Support Engineer I, Ops Tech Solutions (OTS) role. Each phase builds on the previous one, progressing from foundational concepts to advanced specialization.

---

## Phase 1: Networking Fundamentals

**Focus:** Build core understanding of how networks function at each layer.

**Target Courses:**
- Cisco Network Fundamentals Specialization
- Cisco Solutions and Networking Fundamentals Specialization

**Skills Developed:**
- OSI and TCP/IP model understanding
- IP addressing and subnetting
- Ethernet switching and VLAN concepts
- Basic routing principles
- Network device identification and CLI awareness

**Expected Artifacts:**
- Course notes for each module
- Lab screenshots (topology diagrams, CLI outputs)
- Troubleshooting runbook exercises
- Completion certificates

**OTS Readiness:** Enables basic network troubleshooting and structured diagnostic methodology.

---

## Phase 2: Advanced Networking

**Focus:** Develop professional-level networking knowledge aligned with industry certification.

**Target Courses:**
- Cisco CCNA (200-301) Specialization
- Understanding Cisco Data Center Foundations Specialization

**Skills Developed:**
- Advanced switching (STP, EtherChannel, VLANs)
- Dynamic routing protocols (OSPF, EIGRP)
- IP services (DHCP, DNS, NAT, NTP, SNMP)
- Network security (ACLs, port security)
- Data center infrastructure concepts
- High availability and redundancy design

**Expected Artifacts:**
- Detailed study notes with configuration examples
- Lab screenshots from Packet Tracer or GNS3 exercises
- Additional troubleshooting runbooks covering advanced scenarios
- Completion certificates

**OTS Readiness:** Enables root-cause analysis of complex network issues and effective communication with network engineering teams.

---

## Phase 3: Systems and Security

**Focus:** Develop Linux administration skills and security operations awareness.

**Target Courses:**
- Linux and Private Cloud Administration on IBM Power Systems Specialization
- Cybersecurity Operations Fundamentals Specialization

**Skills Developed:**
- Linux command-line proficiency
- System administration (users, processes, services)
- Log analysis and system monitoring
- Security monitoring and alerting
- Incident response fundamentals
- Endpoint protection concepts

**Expected Artifacts:**
- Linux command reference notes
- Security operations procedure documentation
- System monitoring and log analysis screenshots
- Completion certificates

**OTS Readiness:** Enables systems-level troubleshooting, security-aware operations, and log-based diagnostics.

---

## Phase 4: Automation and Integration

**Focus:** Understand how networks are managed at scale through automation and programmability.

**Target Courses:**
- Network Automation Engineering Fundamentals Specialization

**Skills Developed:**
- Network programmability concepts
- Python scripting basics for network tasks
- REST API interactions
- Configuration management (Ansible)
- Understanding of intent-based networking

**Expected Artifacts:**
- Automation concept notes
- API interaction examples and screenshots
- Configuration management workflow documentation
- Completion certificate

**OTS Readiness:** Enables understanding of automated infrastructure environments and effective collaboration with automation-focused engineering teams.

## Sequencing Rationale

1. **Networking first:** Network troubleshooting is the core OTS responsibility. Foundational knowledge must be solid before advancing.
2. **Depth before breadth:** CCNA-level depth in Phase 2 builds diagnostic confidence for complex scenarios.
3. **Systems and security together:** Linux skills and security awareness complement each other and are used together in operations.
4. **Automation last:** Requires networking fundamentals as prerequisite context. Understanding what is being automated matters before understanding how.
