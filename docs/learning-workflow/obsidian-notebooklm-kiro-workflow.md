# Learning Workflow: Obsidian, NotebookLM, ChatGPT, and Kiro

## Overview

This document explains the toolchain and workflow used to transform raw coursework into polished, original, public-facing documentation in this repository. The workflow is designed to separate private study activity from public portfolio artifacts, ensuring that nothing committed to GitHub contains copied course content or violates intellectual property boundaries.

---

## Workflow Stages

### Stage 1: Obsidian Web Clipper (Private Raw Capture)

**Purpose:** Capture and organize raw course material for personal study.

- Obsidian Web Clipper is used to save course pages, reference material, and personal annotations during study sessions
- All raw captures remain in a private Obsidian vault and are never committed to this repository
- This stage is strictly for personal learning and note-taking efficiency
- Captures include bookmarks, highlighted passages, and initial reactions to course content

**Output:** Private study notes in Obsidian (not published)

### Stage 2: NotebookLM (Source Understanding and Study Layer)

**Purpose:** Deepen comprehension and test understanding of course material.

- NotebookLM is used as an interactive study and tutoring tool to review concepts captured in Stage 1
- Helps identify gaps in understanding, clarify confusing topics, and synthesize information across multiple sources
- Functions as a question-and-answer layer for self-testing before documenting what I have learned
- Supports connecting concepts across courses (e.g., linking DHCP behavior in networking fundamentals to troubleshooting scenarios)

**Output:** Improved conceptual understanding and identified knowledge gaps (not published directly)

### Stage 3: ChatGPT (Transformation and Polish)

**Purpose:** Transform rough study notes into professional documentation artifacts.

- ChatGPT helps convert messy personal notes into polished summaries, structured runbooks, professional reflections, and resume-ready language
- Used for refining technical writing, improving clarity, and ensuring consistent formatting
- Assists with framing course concepts in terms of OTS role relevance
- Helps articulate interview-ready explanations of technical topics

**Output:** Polished drafts ready for repository inclusion (reviewed before committing)

### Stage 4: Kiro (Repository Management and Publication)

**Purpose:** Commit polished, original artifacts to the public GitHub repository.

- Kiro is used to create, update, and manage all files in this repository
- Ensures consistent formatting, file placement, and repository structure
- Manages Git operations (commits, pushes, branch management)
- Maintains alignment between documentation and the repository's stated purpose

**Output:** Public-facing portfolio artifacts on GitHub

## Content Ethics and Copyright Boundaries

This repository adheres to strict boundaries regarding intellectual property and course content ownership:

### Do Not Commit

- **Full Coursera lesson text.** Course material belongs to the content providers (Cisco, Packt, IBM, Red Hat). Reproducing it violates platform terms.
- **Quiz answers or graded assessment content.** Assessment integrity must be maintained for all learners.
- **Copied transcripts.** Video transcripts and lecture notes are proprietary content.
- **Proprietary diagrams or visuals.** Unless created independently, course graphics should not be reproduced.

### Acceptable Content

- **Original summaries.** Concepts explained in my own words, demonstrating understanding rather than copying.
- **Personally created lab notes.** Documentation of my own lab exercises, configurations, and outputs.
- **My own CLI outputs and topology diagrams.** Screenshots of work I performed in lab environments.
- **Certificate proof.** Images of earned certificates (these are intended to be shared).
- **Applied troubleshooting documentation.** Runbooks and procedures written from personal understanding of studied concepts.
- **Reflections and growth logs.** Personal observations about learning progress and skill development.

### Guiding Principle

Every artifact in this repository should demonstrate what I have learned and how I can apply it, not what the course said verbatim. The goal is to show understanding, application, and professional communication, not to reproduce educational content.
